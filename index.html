Comment se servir de Github ?

Ecrivons quelques lignes de Lorem :

Lorem ipsum, dolor sit amet consectetur adipisicing elit. Perspiciatis temporibus expedita minima corporis asperiores! Aliquid laborum dolorem facilis cum impedit delectus aperiam in beatae cupiditate?

  • Je vérifie avec git status.
  • Le terminal me répond :
    On branch main
    Your branch is up to date with 'origin/main'.

    Changes not staged for commit:
    (use "git add ..." to update what will be committed)
    (use "git restore ..." to discard changes in working directory)
    modified: index.html

    no changes added to commit (use "git add" and/or "git commit -a")

  • Ce qui veut dire :
    Sur la branche principale
    Votre branche est à jour avec « origin/main ».

    Modifications non validées pour la validation :
    (utilisez « git add… » pour mettre à jour ce qui sera validé)
    (utilisez « git restore… » pour annuler les modifications dans le répertoire de travail)
    modifié : index.html

    Aucune modification ajoutée à la validation (utilisez « git add » et/ou « git commit -a »).

  • 1re étape : Je tape git add index.html
  • Le terminal me répond tout en bas : Pas encore validé (en attente)
  • 2e étape : Photo du fichier à envoyer avec git commit -m "Premier essai dans Hello"
  • Je tape git log pour connaître l'historique du projet.
  • Deux commits sont listés du plus récent au plus ancien avec leur message : le 1er indique "Premier essai dans Hello", le 2e "Ajout du README - souffle d'introduction" (fait avec Copilote).
  • J'arrête là et je remets à jour "Hello" sur github distant.
